# Design System  
A focused, minimal dark interface for a developer‑productivity environment.  
Inspired by civic‑tech clarity and chronological information design seen in modern public‑sector digital work.

## Overview
DevFocus Dark is built for high‑density workflows where clarity, speed, and low visual noise matter.  
The system emphasizes:
- Minimal surfaces  
- Strong hierarchy  
- High‑contrast typography  
- Predictable interaction patterns  
- A calm, utilitarian aesthetic suitable for long coding sessions

## Colors
- **Primary** `#2665fd` — Calls to action, active states, high‑priority interactive elements  
- **Secondary** `#475569` — Supporting UI, chips, secondary actions  
- **Surface** `#0b1326` — Backgrounds, panels, shells  
- **On‑surface** `#dae2fd` — Primary text, icons, and essential information  
- **Error** `#ffb4ab` — Validation errors, destructive actions, alerts

### Color Usage Principles
- Primary should be used sparingly to maintain focus  
- Surfaces should layer subtly without shadows  
- Text must maintain at least a **4:1 contrast ratio** for accessibility

## Typography
- **Headlines** — Inter, semi‑bold  
- **Body** — Inter, regular, 14–16px  
- **Labels** — Inter, medium, 12px, uppercase for section headers  
- Line‑height: 1.4–1.6 for readability on dark backgrounds

## Components

### Buttons
- 8px rounded corners  
- Primary: solid brand blue  
- Secondary: surface‑contrast outline  
- Hover: subtle lightening, no elevation

### Inputs
- 1px border  
- Slightly lighter surface‑variant background  
- Focus ring uses primary color at 2px  
- Labels always visible (no floating labels)

### Cards
- No elevation or drop shadows  
- Defined by border + surface contrast  
- Ideal for grouping related content or chronological lists

### Navigation
- Vertical sidebar recommended for dense developer tools  
- Active item uses primary color indicator  
- Icons monochrome, relying on on‑surface color

### Tables & Data Views
- Dense layout with 8px padding  
- Row hover: subtle surface shift  
- Headings semi‑bold, uppercase optional  
- Ideal for chronological or project‑based listings (inspired by studio work archives) 

## Interaction Guidelines
- Motion minimal and utilitarian  
- State changes should be instant and clearly visible  
- Avoid decorative animations  
- Maintain consistent spacing increments (4px grid)

## Do's and Don'ts
### Do
- Use primary color only for the most important action  
- Maintain consistent corner radii across components  
- Keep layouts clean and information‑dense  
- Ensure all text meets 4:1 contrast ratio  

### Don't
- Mix sharp and rounded corners in the same view  
- Overuse color or elevation  
- Add unnecessary ornamentation  
- Reduce contrast for aesthetic reasons

## Tone & Brand
DevFocus Dark should feel:
- Focused  
- Technical  
- Calm  
- Purpose‑driven  

It aligns with modern public‑sector digital design values—clarity, accessibility, and trustworthiness—similar to the structured, chronological presentation of work seen in National Design Studio's portfolio.
